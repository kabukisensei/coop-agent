# /annotate

Use the `coop-workflow` skill.

I will give you review feedback as **Markdown annotations** — a list keyed to
sections, files, or line ranges (e.g. `glossary.md: define "gross margin"` or
`fact_sales.sql:42: use the shared date dimension`).

Apply **only** the changes I annotate. For each one:

1. Read the referenced location before editing.
2. Make the smallest edit that satisfies the annotation — nothing beyond it.
3. Note anything an annotation would break, and pause instead of guessing.

Stay read-only until I've given the annotations; then present a PLAN of the
edits and get approval before writing. Never commit source.
